# LearnOpenGL - 확산 조도
IBL(이미지 기반 조명)은 기존의 직접 분석 조명 방식과는 달리 물체를 조명하는 여러 기법을 총칭하는 용어입니다. [이전의](https://learnopengl.com/PBR/Lighting) 이 장에서는 주변 환경을 하나의 거대한 광원으로 취급함으로써 조명을 구현하는 방법을 다룹니다. 일반적으로 실제 세계에서 가져오거나 3D 장면에서 생성한 큐브맵 환경 맵을 조작하여 조명 방정식에 직접 사용할 수 있도록 합니다. 즉, 각 큐브맵 텍셀을 광원으로 간주하는 것입니다. 이렇게 하면 환경의 전반적인 조명과 느낌을 효과적으로 포착하여 객체가 환경에 더 자연스럽게 어우러지도록 할 수 있습니다.

이미지 기반 조명 알고리즘은 주변 환경의 조명을 포착하기 때문에, 입력값은 주변 조명의 보다 정밀한 형태, 심지어는 전역 조명의 대략적인 근사치로 간주됩니다. 이러한 이유로 이미지 기반 조명(IBL)은 물리 기반 렌더링(PBR)에 있어 중요한 의미를 지니는데, 환경 조명을 고려할 때 객체가 훨씬 더 물리적으로 정확하게 표현되기 때문입니다.

PBR 시스템에 IBL을 도입하기 위해 반사율 방정식을 다시 한번 간단히 살펴보겠습니다.

\\\[ L\_o(p,\\omega\_o) = \\int\\limits\_{\\Omega} (k\_d\\frac{c}{\\pi} + k\_s\\frac{DFG}{4(\\omega\_o \\cdot n)(\\omega\_i \\cdot n)}) L\_i(p,\\omega\_i) n \\cdot \\omega\_i d\\omega\_i \\\]

앞서 설명했듯이, 우리의 주요 목표는 반구 Ω에 걸쳐 들어오는 모든 빛의 방향 ωi의 적분을 구하는 것입니다. 이전 장에서 적분을 풀 때는 적분에 영향을 미치는 몇 가지 빛의 방향 ωi를 정확히 알고 있었기 때문에 쉬웠습니다. 하지만 이번에는 주변 환경에서 들어오는 **모든** 빛의 방향 ωi가 잠재적으로 복사량을 가질 수 있으므로 적분을 푸는 것이 더 복잡해집니다. 따라서 적분을 풀기 위해서는 두 가지 주요 조건을 충족해야 합니다.

*   임의의 방향 벡터 \\(w\_i\\)가 주어졌을 때 장면의 휘도를 추출할 수 있는 방법이 필요합니다.
*   적분 계산은 빠르고 실시간으로 이루어져야 합니다.

첫 번째 요구 사항은 비교적 쉽습니다. 이미 언급했듯이 환경이나 장면의 조도를 표현하는 한 가지 방법은 (처리된) 환경 큐브맵을 사용하는 것입니다. 이러한 큐브맵이 주어지면 큐브맵의 각 텍셀을 하나의 발광 광원으로 시각화할 수 있습니다. 임의의 방향 벡터 \\(w\_i\\)로 이 큐브맵을 샘플링하면 해당 방향에서 오는 장면의 복사휘도를 얻을 수 있습니다.

임의의 방향 벡터 \\(w\_i\\)가 주어졌을 때 장면의 복사휘도를 구하는 것은 다음과 같이 간단합니다.

```

vec3 radiance = texture(_cubemapEnvironment, w_i).rgb;  

```


하지만 적분을 풀려면 환경 맵을 한 방향뿐만 아니라 반구(Ω) 전체에 걸쳐 가능한 모든 방향(w_i)에서 샘플링해야 하므로 각 프래그먼트 셰이더 호출마다 너무 많은 비용이 소요됩니다. 적분을 보다 효율적으로 풀기 위해서는 대부분의 계산을 미리 처리하거나 사전 계산해야 합니다. 이를 위해서는 반사율 방정식에 대해 좀 더 자세히 살펴봐야 합니다.

\\\[ L\_o(p,\\omega\_o) = \\int\\limits\_{\\Omega} (k\_d\\frac{c}{\\pi} + k\_s\\frac{DFG}{4(\\omega\_o \\cdot n)(\\omega\_i \\cdot n)}) L\_i(p,\\omega\_i) n \\cdot \\omega\_i d\\omega\_i \\\]

반사율 방정식을 자세히 살펴보면 BRDF의 확산 반사(k_d) 항과 정반사(k_s) 항이 서로 독립적이므로 적분을 두 부분으로 나눌 수 있습니다.

\\\[ L\_o(p,\\omega\_o) = \\int\\limits\_{\\Omega} (k\_d\\frac{c}{\\pi}) L\_i(p,\\omega\_i) n \\cdot \\omega\_i d\\omega\_i + \\int\\limits\_{\\Omega} (k\_s\\frac{DFG}{4(\\omega\_o \\cdot n)(\\omega\_i \\cdot n)}) L\_i(p,\\omega\_i) n \\cdot \\omega\_i d\\omega\_i \\\]

적분을 두 부분으로 나누면 확산항과 반사항을 각각 개별적으로 살펴볼 수 있습니다. 이 장에서는 확산 적분에 초점을 맞춥니다.

확산 적분을 자세히 살펴보면 확산 램버트 항은 상수항(색상 c, 굴절률 k_d, π는 적분 전체에 걸쳐 일정함)이며 적분 변수에 의존하지 않는다는 것을 알 수 있습니다. 따라서 상수항을 확산 적분 밖으로 옮길 수 있습니다.

\\\[ L\_o(p,\\omega\_o) = k\_d\\frac{c}{\\pi} \\int\\limits\_{\\Omega} L\_i(p,\\omega\_i) n \\cdot \\omega\_i d\\omega\_i \\\]

이를 통해 (p가 환경 맵의 중심에 있다고 가정할 때) \\(w\_i\\)에만 의존하는 적분을 얻을 수 있습니다. 이 정보를 바탕으로, 각 샘플 방향(또는 텍셀) \\(w\_o\\)에 확산 적분의 결과를 컨볼루션을 통해 저장하는 새로운 큐브맵을 계산하거나 미리 계산할 수 있습니다.

컨볼루션은 데이터 세트의 각 항목에 대해 다른 모든 항목을 고려하여 연산을 적용하는 것입니다. 여기서 데이터 세트는 장면의 복사 휘도 또는 환경 맵입니다. 따라서 큐브맵의 모든 샘플 방향에 대해 반구(Ω) 상의 다른 모든 샘플 방향을 고려합니다.

환경 맵을 컨볼루션하기 위해, 각 출력 방향(w_o)에 대한 적분을 계산합니다. 이를 위해 반구(Ω) 전체에 걸쳐 다수의 방향(w_i)을 이산적으로 샘플링하고, 이들의 복사량을 평균화합니다. 샘플 방향(w_i)을 구성하는 반구는 컨볼루션을 수행하는 출력 방향(w_o)을 향하도록 정렬됩니다.

![PBR 조도 맵을 위해 반구에 큐브맵을 합성합니다.](https://learnopengl.com/img/pbr/ibl_hemisphere_sample.png)

각 샘플 방향 \\(w\_o\\)에 대한 적분 결과를 저장하는 이 사전 계산된 큐브맵은 방향 \\(w\_o\\)를 따라 정렬된 표면에 닿는 장면의 모든 간접 확산광의 사전 계산된 합으로 생각할 수 있습니다. 이러한 큐브맵은 복사조도 맵으로 알려져 있는데, 합성된 큐브맵을 통해 장면의 (사전 계산된) 복사조도를 임의의 방향 \\(w\_o\\)에서 직접 샘플링할 수 있기 때문입니다.

복사휘도 방정식은 위치 \\(p\\)에도 의존하는데, 우리는 이 위치를 조도 맵의 중심으로 가정했습니다. 이는 모든 확산 간접광이 단일 환경 맵에서 나와야 한다는 것을 의미하며, 특히 실내에서 현실감을 해칠 수 있습니다. 렌더링 엔진은 장면 전체에 반사 프로브를 배치하여 이 문제를 해결합니다. 각 반사 프로브는 주변의 조도 맵을 자체적으로 계산합니다. 이렇게 하면 위치 \\(p\\)에서의 조도(및 복사휘도)는 가장 가까운 반사 프로브들 사이에서 보간된 조도가 됩니다. 여기서는 환경 맵을 항상 중심에서 샘플링한다고 가정합니다.

아래는 큐브맵 환경 맵과 그 결과로 생성된 조도 맵의 예시입니다(자료 제공: ). [파동 엔진](http://www.indiedb.com/features/using-image-based-lighting-ibl)장면의 모든 방향(w\_o\)에 대한 휘도를 평균화합니다.

![큐브맵 환경 맵을 컨볼루션하는 효과.](https://learnopengl.com/img/pbr/ibl_irradiance.png)

각 큐브맵 텍셀에 (w_o 방향으로) 합성 결과를 저장함으로써, 조도 맵은 주변 환경의 평균적인 색상이나 조명을 나타내는 것처럼 표시됩니다. 이 환경 맵에서 임의의 방향을 샘플링하면 해당 방향의 장면 조도를 얻을 수 있습니다.

PBR과 HDR
-----------

우리는 이전에 그것에 대해 간략하게 언급했습니다. [이전의](https://learnopengl.com/PBR/Lighting) 이 장에서는 PBR 파이프라인에서 장면 조명의 높은 동적 범위를 고려하는 것이 매우 중요하다고 강조합니다. PBR은 대부분의 입력값을 실제 물리적 속성과 측정값에 기반하므로, 입력되는 광원 값을 물리적 값과 최대한 일치시키는 것이 중요합니다. 각 광원의 복사속을 추측하든, 실제 값을 사용하든 마찬가지입니다. [직접적인 물리적 등가물](https://en.wikipedia.org/wiki/Lumen_\(unit\))단순한 전구와 태양의 차이는 어느 쪽이든 상당합니다. 전문적인 지식 없이는... [HDR](https://learnopengl.com/Advanced-Lighting/HDR) 렌더링 환경에서는 각 광원의 상대적인 강도를 정확하게 지정하는 것이 불가능합니다.

PBR과 HDR은 밀접한 관련이 있지만, 이미지 기반 조명과는 어떤 관계가 있을까요? 이전 장에서 PBR을 HDR 환경에서 구현하는 것이 비교적 쉽다는 것을 살펴보았습니다. 하지만 이미지 기반 조명에서는 환경 큐브맵의 색상 값을 기반으로 환경의 간접광 강도를 결정하기 때문에, 조명의 고해상도(HDR) 데이터를 환경 맵에 저장하는 방법이 필요합니다.

지금까지 우리가 큐브맵으로 사용해 온 환경 맵은 다음과 같습니다. [스카이박스](https://learnopengl.com/Advanced-OpenGL/Cubemaps) 예를 들어, 일부 이미지는 저해상도(LDR)입니다. 우리는 개별 얼굴 이미지에서 색상 값을 직접 사용했으며, 그 범위는 0.5~1.5 사이입니다. `0.0` 그리고 `1.0`그리고 그것들을 있는 그대로 처리했습니다. 이는 시각적 출력에는 문제가 없을 수 있지만, 물리적 입력 매개변수로 사용할 경우에는 제대로 작동하지 않습니다.

### Radiance HDR 파일 형식

라디언스 파일 형식을 입력하십시오. 라디언스 파일 형식(다음과 함께) `.hdr` 확장 기능은 6개의 모든 면을 포함하는 전체 큐브맵을 부동 소수점 데이터로 저장합니다. 이를 통해 큐브맵의 경계를 벗어난 색상 값을 지정할 수 있습니다. `0.0` 에게 `1.0` 조명에 정확한 색상 강도를 부여하기 위한 범위입니다. 또한 이 파일 형식은 각 부동 소수점 값을 채널당 32비트 값이 아닌, 색상의 알파 채널을 지수로 사용하여 채널당 8비트로 저장하는 독창적인 방식을 사용합니다(이로 인해 정밀도가 다소 손실됩니다). 이 방식은 상당히 효과적이지만, 파싱 프로그램에서 각 색상을 해당 부동 소수점 값으로 다시 변환해야 합니다.

다양한 곳에서 무료로 구할 수 있는 Radiance HDR 환경 맵이 꽤 많습니다. [sIBL 아카이브](http://www.hdrlabs.com/sibl/archive.html) 아래에서 그 예시를 보실 수 있습니다.

![정사각도 지도의 예시](https://learnopengl.com/img/pbr/ibl_hdr_radiance.png)

이 이미지는 왜곡되어 보이고 이전에 보았던 환경 맵의 6개 개별 큐브맵 면이 보이지 않기 때문에 예상했던 것과 다를 수 있습니다. 이 환경 맵은 구체에서 평면으로 투영되어 환경을 등거리 직사각형 맵이라고 하는 단일 이미지에 더 쉽게 저장할 수 있도록 만들어졌습니다. 하지만 이로 인해 대부분의 시각적 해상도가 수평 방향에 저장되고 상하 방향의 해상도는 상대적으로 떨어지는 단점이 있습니다. 그러나 대부분의 렌더링 엔진에서 흥미로운 조명과 주변 환경은 대부분 수평 방향에 나타나기 때문에 이는 적절한 절충안입니다.

### HDR 및 stb_image.h

Radiance HDR 이미지를 직접 불러오려면 몇 가지 지식이 필요합니다. [파일 형식](http://radsite.lbl.gov/radiance/refer/Notes/picture_format.html) 그리 어렵지는 않지만, 그럼에도 불구하고 번거롭습니다. 다행히도, 널리 사용되는 원 헤더 라이브러리가 있습니다. [stb_image.h](https://github.com/nothings/stb/blob/master/stb_image.h) Radiance HDR 이미지를 부동 소수점 배열로 직접 로드하는 기능을 지원하며, 이는 저희 요구 사항에 완벽하게 부합합니다. `stb_image` 프로젝트에 추가된 기능을 사용하면 HDR 이미지를 로드하는 것이 이제 다음과 같이 간단해집니다.

```

#include "stb_image.h"
[...]

stbi_set_flip_vertically_on_load(true);
int width, height, nrComponents;
float *data = stbi_loadf("newport_loft.hdr", &width, &height, &nrComponents, 0);
unsigned int hdrTexture;
if (data)
{
    glGenTextures(1, &hdrTexture);
    glBindTexture(GL_TEXTURE_2D, hdrTexture);
    glTexImage2D(GL_TEXTURE_2D, 0, GL_RGB16F, width, height, 0, GL_RGB, GL_FLOAT, data); 

    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_S, GL_CLAMP_TO_EDGE);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_T, GL_CLAMP_TO_EDGE);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MIN_FILTER, GL_LINEAR);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MAG_FILTER, GL_LINEAR);

    stbi_image_free(data);
}
else
{
    std::cout << "Failed to load HDR image." << std::endl;
}  

```


`stb_image.h` HDR 값을 자동으로 부동 소수점 값 목록으로 매핑합니다. 기본적으로 채널당 32비트, 색상당 3개 채널을 사용합니다. 이 정보만으로 직사각형 형태의 HDR 환경 맵을 2D 부동 소수점 텍스처에 저장할 수 있습니다.

### 등거리 사각형 맵에서 큐브맵으로

등거리 사각형 지도를 환경 탐색에 직접 사용할 수도 있지만, 이러한 작업은 상대적으로 비용이 많이 들 수 있으므로 큐브맵에서 직접 샘플링하는 것이 성능 면에서 더 유리합니다. 따라서 이 장에서는 먼저 등거리 사각형 이미지를 큐브맵으로 변환하여 추가 처리를 진행합니다. 이 과정에서 등거리 사각형 지도를 3D 환경 지도처럼 샘플링하는 방법도 보여주는데, 어떤 방법을 사용할지는 사용자의 선택에 달려 있습니다.

정사각 투영 이미지를 큐브맵으로 변환하려면 (단위) 정육면체를 렌더링하고 정사각 투영 이미지를 정육면체의 모든 면에 내부에서 투영하여 각 면의 이미지를 6개씩 추출하여 큐브맵의 한 면으로 사용합니다. 이 정육면체의 정점 셰이더는 단순히 정육면체를 있는 그대로 렌더링하고 해당 위치 정보를 3D 샘플 벡터로 프래그먼트 셰이더에 전달합니다.

```

#version 330 core
layout (location = 0) in vec3 aPos;

out vec3 localPos;

uniform mat4 projection;
uniform mat4 view;

void main()
{
    localPos = aPos;  
    gl_Position =  projection * view * vec4(localPos, 1.0);
}

```


프래그먼트 셰이더에서는 마치 정사각 투영 지도를 정사각의 각 면에 깔끔하게 접어 넣은 것처럼 정사각의 각 부분을 색칠합니다. 이를 위해 정사각의 로컬 위치에서 보간된 프래그먼트의 샘플 방향을 가져온 다음, 이 방향 벡터와 삼각법(구면 좌표계를 직교 좌표계로 변환)을 사용하여 정사각 투영 지도를 마치 정사각 투영 지도 자체인 것처럼 샘플링합니다. 결과는 정사각 투영 지도의 각 면에 직접 저장되며, 이 과정이면 충분합니다.

```

#version 330 core
out vec4 FragColor;
in vec3 localPos;

uniform sampler2D equirectangularMap;

const vec2 invAtan = vec2(0.1591, 0.3183);
vec2 SampleSphericalMap(vec3 v)
{
    vec2 uv = vec2(atan(v.z, v.x), asin(v.y));
    uv *= invAtan;
    uv += 0.5;
    return uv;
}

void main()
{		
    vec2 uv = SampleSphericalMap(normalize(localPos)); // make sure to normalize localPos
    vec3 color = texture(equirectangularMap, uv).rgb;
    
    FragColor = vec4(color, 1.0);
}


```


HDR 등거리 직사각형 맵을 사용하여 장면 중앙에 큐브를 렌더링하면 다음과 같은 결과가 나타납니다.

![큐브맵으로 변환된 등거리 사각형 지도를 OpenGL로 렌더링한 결과입니다.](https://learnopengl.com/img/pbr/ibl_equirectangular_projection.png)

이는 우리가 정사각 투영 이미지를 정육면체 모양에 효과적으로 매핑했음을 보여주지만, 원본 HDR 이미지를 큐브맵 텍스처로 변환하는 데는 아직 도움이 되지 않습니다. 이를 위해서는 동일한 정육면체를 6번 렌더링하고, 각 면을 개별적으로 살펴보면서 그 시각적 결과를 기록해야 합니다. [프레임버퍼](https://learnopengl.com/Advanced-OpenGL/Framebuffers) 물체:

```

unsigned int captureFBO, captureRBO;
glGenFramebuffers(1, &captureFBO);
glGenRenderbuffers(1, &captureRBO);

glBindFramebuffer(GL_FRAMEBUFFER, captureFBO);
glBindRenderbuffer(GL_RENDERBUFFER, captureRBO);
glRenderbufferStorage(GL_RENDERBUFFER, GL_DEPTH_COMPONENT24, 512, 512);
glFramebufferRenderbuffer(GL_FRAMEBUFFER, GL_DEPTH_ATTACHMENT, GL_RENDERBUFFER, captureRBO);  

```


물론, 그 다음에는 해당 큐브맵 색상 텍스처를 생성하고, 6개의 면 각각에 대해 메모리를 미리 할당합니다.

```

unsigned int envCubemap;
glGenTextures(1, &envCubemap);
glBindTexture(GL_TEXTURE_CUBE_MAP, envCubemap);
for (unsigned int i = 0; i < 6; ++i)
{
    // note that we store each face with 16 bit floating point values
    glTexImage2D(GL_TEXTURE_CUBE_MAP_POSITIVE_X + i, 0, GL_RGB16F, 
                 512, 512, 0, GL_RGB, GL_FLOAT, nullptr);
}
glTexParameteri(GL_TEXTURE_CUBE_MAP, GL_TEXTURE_WRAP_S, GL_CLAMP_TO_EDGE);
glTexParameteri(GL_TEXTURE_CUBE_MAP, GL_TEXTURE_WRAP_T, GL_CLAMP_TO_EDGE);
glTexParameteri(GL_TEXTURE_CUBE_MAP, GL_TEXTURE_WRAP_R, GL_CLAMP_TO_EDGE);
glTexParameteri(GL_TEXTURE_CUBE_MAP, GL_TEXTURE_MIN_FILTER, GL_LINEAR);
glTexParameteri(GL_TEXTURE_CUBE_MAP, GL_TEXTURE_MAG_FILTER, GL_LINEAR);

```


그다음 남은 작업은 정사각 2D 텍스처를 큐브맵 면에 적용하는 것입니다.

자세한 내용은 코드에 이미 설명되어 있으므로 여기서는 생략하겠습니다. [프레임버퍼](https://learnopengl.com/Advanced-OpenGL/Framebuffers) 그리고 [점 그림자](https://learnopengl.com/Advanced-Lighting/Shadows/Point-Shadows) 여러 챕터로 구성되어 있지만, 핵심은 큐브의 각 면을 향하는 6개의 서로 다른 뷰 매트릭스를 설정하고, 시야각(FOV)을 가진 투영 매트릭스를 설정하는 것입니다. `90` 얼굴 전체를 포착하기 위해 각도를 조정하고, 큐브를 6번 렌더링하여 결과를 부동 소수점 프레임 버퍼에 저장합니다.

```

glm::mat4 captureProjection = glm::perspective(glm::radians(90.0f), 1.0f, 0.1f, 10.0f);
glm::mat4 captureViews[] = 
{
   glm::lookAt(glm::vec3(0.0f, 0.0f, 0.0f), glm::vec3( 1.0f,  0.0f,  0.0f), glm::vec3(0.0f, -1.0f,  0.0f)),
   glm::lookAt(glm::vec3(0.0f, 0.0f, 0.0f), glm::vec3(-1.0f,  0.0f,  0.0f), glm::vec3(0.0f, -1.0f,  0.0f)),
   glm::lookAt(glm::vec3(0.0f, 0.0f, 0.0f), glm::vec3( 0.0f,  1.0f,  0.0f), glm::vec3(0.0f,  0.0f,  1.0f)),
   glm::lookAt(glm::vec3(0.0f, 0.0f, 0.0f), glm::vec3( 0.0f, -1.0f,  0.0f), glm::vec3(0.0f,  0.0f, -1.0f)),
   glm::lookAt(glm::vec3(0.0f, 0.0f, 0.0f), glm::vec3( 0.0f,  0.0f,  1.0f), glm::vec3(0.0f, -1.0f,  0.0f)),
   glm::lookAt(glm::vec3(0.0f, 0.0f, 0.0f), glm::vec3( 0.0f,  0.0f, -1.0f), glm::vec3(0.0f, -1.0f,  0.0f))
};

// convert HDR equirectangular environment map to cubemap equivalent
equirectangularToCubemapShader.use();
equirectangularToCubemapShader.setInt("equirectangularMap", 0);
equirectangularToCubemapShader.setMat4("projection", captureProjection);
glActiveTexture(GL_TEXTURE0);
glBindTexture(GL_TEXTURE_2D, hdrTexture);

glViewport(0, 0, 512, 512); // don't forget to configure the viewport to the capture dimensions.
glBindFramebuffer(GL_FRAMEBUFFER, captureFBO);
for (unsigned int i = 0; i < 6; ++i)
{
    equirectangularToCubemapShader.setMat4("view", captureViews[i]);
    glFramebufferTexture2D(GL_FRAMEBUFFER, GL_COLOR_ATTACHMENT0, 
                           GL_TEXTURE_CUBE_MAP_POSITIVE_X + i, envCubemap, 0);
    glClear(GL_COLOR_BUFFER_BIT | GL_DEPTH_BUFFER_BIT);

    renderCube(); // renders a 1x1 cube
}
glBindFramebuffer(GL_FRAMEBUFFER, 0);  

```


프레임버퍼의 컬러 첨부 파일을 가져와 큐브맵의 각 면에 대해 텍스처 대상을 바꿔가며 장면을 큐브맵의 한 면에 직접 렌더링합니다. 이 루틴이 완료되면(한 번만 수행하면 됨) `envCubemap`이라는 큐브맵이 생성되어 원본 HDR 이미지의 환경 버전이 됩니다.

큐브맵을 테스트하기 위해 주변에 큐브맵을 표시하는 아주 간단한 스카이박스 셰이더를 작성해 보겠습니다.

```

#version 330 core
layout (location = 0) in vec3 aPos;

uniform mat4 projection;
uniform mat4 view;

out vec3 localPos;

void main()
{
    localPos = aPos;

    mat4 rotView = mat4(mat3(view)); // remove translation from the view matrix
    vec4 clipPos = projection * rotView * vec4(localPos, 1.0);

    gl_Position = clipPos.xyww;
}

```


참고하십시오 `xyww` 여기서 사용하는 트릭은 렌더링된 큐브 조각의 깊이 값이 항상 특정 값으로 유지되도록 합니다. `1.0`최대 깊이 값은 다음과 같이 설명됩니다. [큐브맵](https://learnopengl.com/Advanced-OpenGL/Cubemaps) 해당 장을 참조하십시오. 깊이 비교 함수를 GL_LEQUAL로 변경해야 한다는 점에 유의하십시오.

```

glDepthFunc(GL_LEQUAL);  

```


프래그먼트 셰이더는 큐브의 로컬 프래그먼트 위치를 사용하여 큐브맵 환경 맵을 직접 샘플링합니다.

```

#version 330 core
out vec4 FragColor;

in vec3 localPos;
  
uniform samplerCube environmentMap;
  
void main()
{
    vec3 envColor = texture(environmentMap, localPos).rgb;
    
    envColor = envColor / (envColor + vec3(1.0));
    envColor = pow(envColor, vec3(1.0/2.2)); 
  
    FragColor = vec4(envColor, 1.0);
}

```


우리는 샘플링할 올바른 방향 벡터에 직접 대응하는 보간된 정점 큐브 위치를 사용하여 환경 맵을 샘플링합니다. 카메라의 변환 구성 요소는 무시되므로 큐브 위에 이 셰이더를 렌더링하면 환경 맵이 움직이지 않는 배경으로 나타납니다. 또한 환경 맵의 HDR 값을 기본 LDR 프레임 버퍼로 직접 출력하기 때문에 색상 값을 적절하게 톤 매핑해야 합니다. 게다가 거의 모든 HDR 맵은 기본적으로 선형 색 공간을 사용하므로 톤 매핑을 적용해야 합니다. [감마 보정](https://learnopengl.com/Advanced-Lighting/Gamma-Correction) 기본 프레임버퍼에 쓰기 전에.

이제 샘플링된 환경 지도를 이전에 렌더링된 구체 위에 렌더링하면 다음과 같은 모습이 될 것입니다.

![변환된 큐브맵을 스카이박스로 렌더링합니다.](https://learnopengl.com/img/pbr/ibl_hdr_environment_mapped.png)

음... 여기까지 오는 데 꽤 많은 준비 과정이 필요했지만, HDR 환경 맵을 읽어들이고, 이를 등거리 사각형 매핑에서 큐브맵으로 변환한 다음, HDR 큐브맵을 스카이박스로 장면에 렌더링하는 데 성공했습니다. 더 나아가, 큐브맵의 6개 면 모두에 렌더링하는 간단한 시스템도 구축했는데, 이는 환경 맵을 컨볼루션할 때 다시 필요할 것입니다. 전체 변환 과정의 소스 코드는 여기에서 확인할 수 있습니다. [여기](https://learnopengl.com/code_viewer_gh.php?code=src/6.pbr/2.1.1.ibl_irradiance_conversion/ibl_irradiance_conversion.cpp).

큐브맵 컨볼루션
-------------------

이 장의 서두에서 설명했듯이, 우리의 주요 목표는 큐브맵 환경 맵 형태의 장면 조도가 주어졌을 때 모든 확산 간접 조명에 대한 적분을 계산하는 것입니다. 특정 방향의 장면 복사휘도 \\(L(p, w\_i)\\)는 방향 \\(w\_i\\)로 HDR 환경 맵을 샘플링하여 얻을 수 있다는 것을 알고 있습니다. 적분을 계산하기 위해서는 각 프래그먼트에 대해 반구 \\(\\Omega\\) 내의 모든 가능한 방향에서 장면의 복사휘도를 샘플링해야 합니다.

하지만 Ω 내의 모든 가능한 방향에서 환경의 조명을 샘플링하는 것은 계산상 불가능합니다. 이론적으로 가능한 방향의 수는 무한하기 때문입니다. 그러나 유한한 수의 방향 또는 샘플을 사용하여 방향의 수를 근사화할 수 있습니다. 이러한 샘플은 반구 내에서 균일하게 또는 무작위로 배치되며, 조도를 상당히 정확하게 근사할 수 있습니다. 즉, 적분 ∫₀ᵀ를 이산적으로 푸는 것입니다.

하지만 모든 조각에 대해 실시간으로 이 작업을 수행하는 것은 여전히 ​​너무 비용이 많이 듭니다. 제대로 된 결과를 얻으려면 샘플 수가 상당히 많아야 하기 때문입니다. 따라서 이 작업을 미리 계산하고자 합니다. 반구의 방향에 따라 조도를 측정하는 위치가 결정되므로, 모든 방향(w\_o)을 중심으로 가능한 모든 반구 방향에 대한 조도를 미리 계산할 수 있습니다.

\\\[ L\_o(p,\\omega\_o) = k\_d\\frac{c}{\\pi} \\int\\limits\_{\\Omega} L\_i(p,\\omega\_i) n \\cdot \\omega\_i d\\omega\_i \\\]

라이팅 패스에서 임의의 방향 벡터 \\(w\_i\\)가 주어지면, 미리 계산된 조도 맵을 샘플링하여 해당 방향 \\(w\_i\\)에서의 총 확산 조도를 얻을 수 있습니다. 프래그먼트 표면에서의 간접 확산(조도) 광량을 결정하기 위해, 해당 표면의 법선을 중심으로 하는 반구에서 총 조도를 가져옵니다. 따라서 장면의 조도를 얻는 것은 다음과 같이 간단합니다.

```

vec3 irradiance = texture(irradianceMap, N).rgb;

```


이제 조도 맵을 생성하려면 큐브맵으로 변환된 환경 조명을 컨볼루션해야 합니다. 각 프래그먼트에 대해 표면의 반구가 법선 벡터 \\(N\\)을 따라 정렬되어 있다고 가정하면, 큐브맵을 컨볼루션하는 것은 \\(N\\)을 따라 정렬된 반구 \\(\\Omega\\) 내의 각 방향 \\(w\_i\\)의 총 평균 복사량을 계산하는 것과 같습니다.

![PBR 조도 맵을 위해 (법선 방향을 중심으로 하는) 반구에 큐브맵을 합성합니다.](https://learnopengl.com/img/pbr/ibl_hemisphere_sample_normal.png)

다행히도, 이 장에서의 번거로운 설정 과정이 모두 헛된 것은 아닙니다. 이제 변환된 큐브맵을 직접 가져와 프래그먼트 셰이더에서 컨볼루션하고, 6개 면 방향 모두에 렌더링하는 프레임버퍼를 사용하여 그 결과를 새로운 큐브맵에 저장할 수 있습니다. 이미 등거리 직사각형 환경 맵을 큐브맵으로 변환하는 방법을 설정했으므로, 동일한 접근 방식을 사용하되 다른 프래그먼트 셰이더를 사용하면 됩니다.

```

const float PI = 3.14159265359;

    // the sample direction equals the hemisphere's orientation 
    vec3 normal = normalize(localPos);
  
    vec3 irradiance = vec3(0.0);
  
    [...] // convolution code

```


environmentMap은 직사각형 형태의 HDR 환경 맵에서 변환된 HDR 큐브맵입니다.

환경 맵을 합성하는 방법은 여러 가지가 있지만, 이 장에서는 샘플 방향을 중심으로 하는 반구(Ω)를 따라 각 큐브맵 텍셀에 대해 고정된 개수의 샘플 벡터를 생성하고 그 결과를 평균화하는 방법을 사용하겠습니다. 고정된 개수의 샘플 벡터는 반구 내부에 균일하게 분포됩니다. 적분은 연속 함수이므로 고정된 개수의 샘플 벡터를 사용하여 함수를 이산적으로 샘플링하는 것은 근사값에 불과하다는 점에 유의하십시오. 사용하는 샘플 벡터의 수가 많을수록 적분을 더 정확하게 근사할 수 있습니다.

반사율 방정식의 적분 \\(\\int\\)은 다루기가 다소 어려운 입체각 \\(dw\\)를 중심으로 이루어집니다. 입체각 \\(dw\\)에 대한 적분 대신, 이에 상응하는 구면 좌표 \\(\\theta\\)와 \\(\\phi\\)에 대한 적분을 수행하겠습니다.

![PBR에 대한 등가 극좌표 방위각 및 경사각으로 입체각을 변환합니다.](https://learnopengl.com/img/pbr/ibl_spherical_integrate.png)

우리는 극좌표 방위각 φ를 사용하여 0에서 2π 사이의 반구 고리 주변을 샘플링하고, 천정 경사각 θ를 사용하여 0에서 1/2π 사이의 반구 고리가 증가하는 부분을 샘플링합니다. 이렇게 하면 업데이트된 반사율 적분값을 얻을 수 있습니다.

\\\[ L\_o(p,\\phi\_o, \\theta\_o) = k\_d\\frac{c}{\\pi} \\int\_{\\phi = 0}^{2\\pi} \\int\_{\\theta = 0}^{\\frac{1}{2}\\pi} L\_i(p,\\phi\_i, \\theta\_i) \\cos(\\theta) \\sin(\\theta) d\\phi d\\theta \\\]

적분을 풀기 위해서는 반구 \\(\\Omega\\) 내에서 고정된 개수의 이산 샘플을 추출하고 그 결과를 평균해야 합니다. 이렇게 하면 적분은 다음과 같은 이산 형태로 변환됩니다. [리만 합](https://en.wikipedia.org/wiki/Riemann_sum) 각 구면 좌표에 대해 각각 \\(n1\\) 및 \\(n2\\)개의 이산 샘플이 주어졌을 때:

\\\[ L\_o(p,\\phi\_o, \\theta\_o) = k\_d \\frac{c\\pi}{n1 n2} \\sum\_{\\phi = 0}^{n1} \\sum\_{\\theta = 0}^{n2} L\_i(p,\\phi\_i, \\theta\_i) \\cos(\\theta) \\sin(\\theta) d\\phi d\\theta \\\]

두 구형 값을 이산적으로 샘플링하기 때문에 각 샘플은 이전 이미지에서처럼 반구의 영역을 근사하거나 평균화합니다. (구형 모양의 일반적인 특성으로 인해) 반구의 이산 샘플 영역은 천정각 θ가 클수록 중심 상단으로 수렴하므로 작아집니다. 이러한 작은 영역을 보정하기 위해 해당 영역을 sin θ로 스케일링하여 가중치를 부여합니다.

적분의 구면 좌표가 주어졌을 때 반구를 이산적으로 샘플링하는 것은 다음과 같은 코드 조각으로 표현됩니다.

```

vec3 irradiance = vec3(0.0);  

vec3 up    = vec3(0.0, 1.0, 0.0);
vec3 right = normalize(cross(up, normal));
up         = normalize(cross(normal, right));

float sampleDelta = 0.025;
float nrSamples = 0.0; 
for(float phi = 0.0; phi < 2.0 * PI; phi += sampleDelta)
{
    for(float theta = 0.0; theta < 0.5 * PI; theta += sampleDelta)
    {
        // spherical to cartesian (in tangent space)
        vec3 tangentSample = vec3(sin(theta) * cos(phi),  sin(theta) * sin(phi), cos(theta));
        // tangent space to world
        vec3 sampleVec = tangentSample.x * right + tangentSample.y * up + tangentSample.z * N; 

        irradiance += texture(environmentMap, sampleVec).rgb * cos(theta) * sin(theta);
        nrSamples++;
    }
}
irradiance = PI * irradiance * (1.0 / float(nrSamples));

```


반구를 횡단하기 위해 고정된 sampleDelta 값을 지정합니다. sampleDelta 값을 줄이거나 늘리면 정확도가 각각 증가하거나 감소합니다.

두 루프 모두에서 구면 좌표를 가져와 3D 직교 좌표계 샘플 벡터로 변환하고, 접선 방향 샘플을 법선 방향을 기준으로 하는 월드 공간으로 변환한 다음, 이 샘플 벡터를 사용하여 HDR 환경 맵을 직접 샘플링합니다. 각 샘플 결과를 조도에 더하고, 마지막으로 전체 샘플 수로 나누어 평균 샘플링 조도를 계산합니다. 샘플링된 색상 값은 스케일링됩니다. `cos(theta)` 빛이 각도가 커질수록 약해지기 때문에 그리고 `sin(theta)` 상위 반구 지역의 표본 면적이 더 작다는 점을 고려하기 위해서입니다.

이제 남은 작업은 이전에 캡처한 envCubemap을 합성할 수 있도록 OpenGL 렌더링 코드를 설정하는 것입니다. 먼저 조도 큐브맵을 생성합니다(이 작업은 렌더링 루프 전에 한 번만 수행하면 됩니다).

```

unsigned int irradianceMap;
glGenTextures(1, &irradianceMap);
glBindTexture(GL_TEXTURE_CUBE_MAP, irradianceMap);
for (unsigned int i = 0; i < 6; ++i)
{
    glTexImage2D(GL_TEXTURE_CUBE_MAP_POSITIVE_X + i, 0, GL_RGB16F, 32, 32, 0, 
                 GL_RGB, GL_FLOAT, nullptr);
}
glTexParameteri(GL_TEXTURE_CUBE_MAP, GL_TEXTURE_WRAP_S, GL_CLAMP_TO_EDGE);
glTexParameteri(GL_TEXTURE_CUBE_MAP, GL_TEXTURE_WRAP_T, GL_CLAMP_TO_EDGE);
glTexParameteri(GL_TEXTURE_CUBE_MAP, GL_TEXTURE_WRAP_R, GL_CLAMP_TO_EDGE);
glTexParameteri(GL_TEXTURE_CUBE_MAP, GL_TEXTURE_MIN_FILTER, GL_LINEAR);
glTexParameteri(GL_TEXTURE_CUBE_MAP, GL_TEXTURE_MAG_FILTER, GL_LINEAR);

```


조도 맵은 주변의 모든 복사량을 균일하게 평균화하기 때문에 고주파 세부 정보가 많지 않습니다. 따라서 맵을 낮은 해상도(32x32)로 저장하고 OpenGL의 선형 필터링이 대부분의 작업을 처리하도록 할 수 있습니다. 다음으로 캡처 프레임 버퍼를 새 해상도로 재조정합니다.

```

glBindFramebuffer(GL_FRAMEBUFFER, captureFBO);
glBindRenderbuffer(GL_RENDERBUFFER, captureRBO);
glRenderbufferStorage(GL_RENDERBUFFER, GL_DEPTH_COMPONENT24, 32, 32);  

```


컨볼루션 셰이더를 사용하여 환경 큐브맵을 캡처했던 방식과 유사하게 환경 맵을 렌더링합니다.

```

irradianceShader.use();
irradianceShader.setInt("environmentMap", 0);
irradianceShader.setMat4("projection", captureProjection);
glActiveTexture(GL_TEXTURE0);
glBindTexture(GL_TEXTURE_CUBE_MAP, envCubemap);

glViewport(0, 0, 32, 32); // don't forget to configure the viewport to the capture dimensions.
glBindFramebuffer(GL_FRAMEBUFFER, captureFBO);
for (unsigned int i = 0; i < 6; ++i)
{
    irradianceShader.setMat4("view", captureViews[i]);
    glFramebufferTexture2D(GL_FRAMEBUFFER, GL_COLOR_ATTACHMENT0, 
                           GL_TEXTURE_CUBE_MAP_POSITIVE_X + i, irradianceMap, 0);
    glClear(GL_COLOR_BUFFER_BIT | GL_DEPTH_BUFFER_BIT);

    renderCube();
}
glBindFramebuffer(GL_FRAMEBUFFER, 0);  

```


이 과정을 마치면 확산 이미지 기반 조명에 직접 사용할 수 있는 사전 계산된 조도 맵을 얻게 됩니다. 환경 맵이 성공적으로 합성되었는지 확인하기 위해 스카이박스의 환경 샘플러로 조도 맵 대신 환경 맵을 사용해 보겠습니다.

![스카이박스 배경으로 PBR 조도 맵을 표시합니다.](https://learnopengl.com/img/pbr/ibl_irradiance_map_background.png)

환경 지도가 심하게 흐릿하게 보인다면 환경 지도를 성공적으로 합성한 것입니다.

PBR 및 간접 조사 조명
------------------------------------

조도 맵은 주변의 모든 간접광으로부터 누적된 반사율 적분의 확산 부분을 나타냅니다. 빛이 직접 광원이 아닌 주변 환경에서 나오기 때문에 확산 및 반사 간접광을 모두 주변광으로 간주하여 이전에 설정했던 상수항을 대체합니다.

먼저, 미리 계산된 조도 맵을 큐브 샘플러로 추가해야 합니다.

```

uniform samplerCube irradianceMap;

```


장면의 모든 간접 확산광을 담고 있는 조도 맵이 주어졌을 때, 프래그먼트에 영향을 미치는 조도를 추출하는 것은 표면 법선을 기준으로 단 하나의 텍스처 샘플만 사용하면 되는 간단한 작업입니다.

```

// vec3 ambient = vec3(0.03);
vec3 ambient = texture(irradianceMap, N).rgb;

```


하지만 간접 조명은 확산광과 정반사광 부분을 모두 포함하고 있으므로(반사율 방정식의 분할 버전에서 보았듯이) 확산광 부분에 적절한 가중치를 부여해야 합니다. 이전 장에서 했던 것과 유사하게 프레넬 방정식을 사용하여 표면의 간접 반사율을 구하고, 이를 통해 굴절(또는 확산) 반사율을 도출합니다.

```

vec3 kS = fresnelSchlick(max(dot(N, V), 0.0), F0);
vec3 kD = 1.0 - kS;
vec3 irradiance = texture(irradianceMap, N).rgb;
vec3 diffuse    = irradiance * albedo;
vec3 ambient    = (kD * diffuse) * ao; 

```


법선 벡터 N을 중심으로 하는 반구 내 모든 방향에서 주변광이 들어오기 때문에 프레넬 응답을 결정하는 단일 중간 벡터가 존재하지 않습니다. 프레넬 효과를 시뮬레이션하기 위해 법선 벡터와 시야 벡터 사이의 각도를 이용하여 프레넬 값을 계산합니다. 하지만 이전에는 표면 거칠기의 영향을 받는 미세 표면 중간 벡터를 프레넬 방정식의 입력값으로 사용했습니다. 현재는 표면 거칠기를 고려하지 않기 때문에 표면의 반사율은 항상 상대적으로 높게 나타납니다. 간접광은 직접광과 동일한 특성을 가지므로 표면이 거칠수록 가장자리에서 반사가 약해질 것으로 예상됩니다. 이 때문에 거친 비금속 표면에서는 간접 프레넬 반사 강도가 실제와 다르게 나타납니다(설명을 위해 약간 과장했습니다).

![표면 거칠기를 고려하지 않은 IBL에 대한 프레넬 방정식.](https://learnopengl.com/img/pbr/lighting_fresnel_no_roughness.png)

우리는 프레넬-슐릭 방정식에 거칠기 항을 도입함으로써 이 문제를 완화할 수 있습니다. [세바스티앙 라가르드](https://seblagarde.wordpress.com/2011/08/17/hello-world/):

```

vec3 fresnelSchlickRoughness(float cosTheta, vec3 F0, float roughness)
{
    return F0 + (max(vec3(1.0 - roughness), F0) - F0) * pow(clamp(1.0 - cosTheta, 0.0, 1.0), 5.0);
}   

```


프레넬 응답을 계산할 때 표면의 거칠기를 고려하면 주변광 코드는 다음과 같습니다.

```

vec3 kS = fresnelSchlickRoughness(max(dot(N, V), 0.0), F0, roughness); 
vec3 kD = 1.0 - kS;
vec3 irradiance = texture(irradianceMap, N).rgb;
vec3 diffuse    = irradiance * albedo;
vec3 ambient    = (kD * diffuse) * ao; 

```


보시다시피, 실제 이미지 기반 조명 계산은 매우 간단하며 큐브맵 텍스처 조회 한 번만 필요합니다. 대부분의 작업은 조도 맵을 사전 계산하거나 합성하는 데 있습니다.

PBR의 초기 장면을 살펴보면 [조명](https://learnopengl.com/PBR/Lighting) 각 구체가 수직 방향으로 금속성 값이 증가하고 수평 방향으로 거칠기 값이 증가하는 이 장에서, 확산 이미지 기반 조명을 추가하면 대략 다음과 같이 보일 것입니다.

![PBR 셰이더에서 사용되는 OpenGL 조도 맵을 컨볼루션한 결과입니다.](https://learnopengl.com/img/pbr/ibl_irradiance_result.png)

금속성 구체는 금속 표면처럼 제대로 보이려면 어느 정도 반사가 필요하기 때문에 여전히 약간 어색해 보입니다. 금속 표면은 확산광을 반사하지 않는데, 현재는 점광원에서만 (아주 약하게) 반사가 나오고 있기 때문입니다. 그럼에도 불구하고, 구체가 주변 환경에 더 자연스럽게 어우러지는 것을 느낄 수 있습니다 (특히 환경 맵을 전환할 때). 표면 반응이 주변 조명에 따라 적절하게 변하기 때문입니다.
