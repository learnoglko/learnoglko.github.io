# LearnOpenGL - 조명
그 안에서 [이전의](https://learnopengl.com/PBR/Theory) 지난 장에서는 사실적인 물리 기반 렌더러를 구현하기 위한 기초를 다졌습니다. 이번 장에서는 앞서 논의한 이론을 직접광원(또는 해석광원)을 사용하는 실제 렌더러로 구현하는 데 집중할 것입니다. 여기서는 점광원, 방향광, 스포트라이트 등을 예로 들 수 있습니다.

이전 장에서 살펴본 최종 반사율 방정식을 다시 한번 살펴보겠습니다.

\\\[ L\_o(p,\\omega\_o) = \\int\\limits\_{\\Omega} (k\_d\\frac{c}{\\pi} + \\frac{DFG}{4(\\omega\_o \\cdot n)(\\omega\_i \\cdot n)}) L\_i(p,\\omega\_i) n \\cdot \\omega\_i d\\omega\_i \\\]

이제 우리는 대부분의 상황을 이해했지만, 여전히 큰 미지수는 장면의 총 복사휘도(L)를 정확히 어떻게 표현할 것인가 하는 점입니다. 컴퓨터 그래픽 분야에서 해석되는 복사휘도(L)는 주어진 입체각(ω)에 걸쳐 광원의 복사속(φ) 또는 광에너지를 측정하는 값입니다. 우리의 경우, 입체각(ω)이 무한히 작다고 가정했는데, 이 경우 복사휘도는 단일 광선 또는 방향 벡터에 걸쳐 광원의 복사속을 측정하는 값입니다.

이러한 지식을 바탕으로, 이전 장에서 습득한 조명 관련 지식을 어떻게 적용할 수 있을까요? 모든 방향으로 동일하게 밝게 빛나는 단일 점광원(광원)이 있다고 가정해 봅시다. 이 점광원의 복사속은 입니다. `(23.47, 21.31, 20.79)` RGB 삼중항으로 변환됩니다. 이 광원의 복사 강도는 모든 나가는 방향 광선에서 복사속과 같습니다. 그러나 표면의 특정 지점(p)을 음영 처리할 때, 표면의 반구(Ω)에 걸쳐 가능한 모든 입사광 방향 중에서 오직 하나의 입사 방향 벡터(w_i)만이 점 광원에서 직접 나옵니다. 장면에는 공간상의 단일 지점으로 가정되는 단 하나의 광원만 있으므로, 다른 모든 가능한 입사광 방향은 표면 지점(p)에서 관측되는 복사량이 0입니다.

![감쇠되지 않은 점 광원의 점 p에서의 복사휘도는 무한히 작은 입체각 Wi 또는 광 방향 벡터 Wi에서만 0이 아닌 값을 갖습니다.](https://learnopengl.com/img/pbr/lighting_radiance_direct.png)

우선, 빛의 감쇠(거리에 따른 빛의 밝기 감소)가 점 광원에 영향을 미치지 않는다고 가정하면, 입사광선의 복사휘도는 광원의 위치에 관계없이 동일합니다(입사각(cosθ)에 따른 복사휘도의 스케일링은 제외). 이는 점 광원이 관측 각도에 관계없이 동일한 복사강도를 가지며, 복사강도를 복사속이라는 상수 벡터로 모델링할 수 있기 때문입니다. `(23.47, 21.31, 20.79)`.

하지만, 복사휘도는 위치 \\(p\\)를 입력값으로 받으며, 실제 점 광원은 빛의 감쇠를 고려하기 때문에 점 광원의 복사 강도는 점 \\(p\\)와 광원 사이의 거리에 비례하여 스케일링됩니다. 그런 다음, 원래 복사휘도 방정식에서 추출한 결과는 표면 법선 \\(n\\)과 입사광 방향 \\(w\_i\\)의 내적으로 스케일링됩니다.

이를 보다 실용적인 관점에서 설명하자면, 직접 점광원의 경우 복사휘도 함수 \\(L\\)은 빛의 색상을 측정하는데, 이는 점 \\(p\\)까지의 거리에 따라 감쇠되고 \\(n \\cdot w\_i\\)로 확대됩니다. 하지만 이는 점 \\(p\\)에 도달하는 단일 광선 \\(w\_i\\)에 대해서만 적용되며, 이 광선은 점 \\(p\\)에서 오는 빛의 방향 벡터와 같습니다. 코드로 표현하면 다음과 같습니다.

```

vec3  lightColor  = vec3(23.47, 21.31, 20.79);
vec3  wi          = normalize(lightPos - fragPos);
float cosTheta    = max(dot(N, Wi), 0.0);
float attenuation = calculateAttenuation(fragPos, lightPos);
vec3  radiance    = lightColor * attenuation * cosTheta;

```


용어만 다를 뿐, 이 코드는 여러분에게 매우 익숙할 것입니다. 지금까지 확산 조명을 구현해 온 방식과 정확히 같습니다. 직접 조명의 경우, 표면의 복사 휘도에는 단 하나의 광원 방향 벡터만 영향을 미치므로, 기존 조명 계산 방식과 유사하게 복사 휘도를 계산합니다.

점광원은 무한히 작고 공간상의 단 하나의 점에 불과하기 때문에 이러한 가정이 성립한다는 점에 유의하십시오. 만약 면적이나 부피를 가진 광원을 모델링한다면, 그 광원의 복사 휘도는 하나 이상의 입사 방향에서 0이 아닌 값을 가질 것입니다.

한 점에서 나오는 다른 유형의 광원에 대해서도 유사한 방식으로 복사휘도를 계산합니다. 예를 들어, 방향성 광원은 감쇠 계수 없이 일정한 \\(w\_i\\) 값을 가집니다. 스포트라이트는 일정한 복사 강도를 가지는 것이 아니라, 스포트라이트의 전방 방향 벡터에 따라 크기가 변하는 복사 강도를 가집니다.

이로써 우리는 표면의 반구(Ω)에 대한 적분(int)으로 다시 돌아가게 됩니다. 단일 표면 지점을 음영 처리하는 동안 모든 광원의 위치를 ​​미리 알고 있으므로 적분을 계산할 필요가 없습니다. 각 광원이 표면의 복사 휘도에 영향을 미치는 단 하나의 광 방향만 가지고 있다는 점을 고려하여 (알려진) 광원의 개수를 직접 사용하여 총 조도를 계산할 수 있습니다. 따라서 직접 광원에 대한 PBR은 기여하는 광원들을 순회하기만 하면 되므로 상대적으로 간단합니다. 나중에 환경 조명을 고려할 때 [아이블](https://learnopengl.com/PBR/IBL/Diffuse-irradiance) 빛이 어느 방향에서든 올 수 있으므로 적분을 고려해야 하는 장들이 있습니다.

PBR 표면 모델
-------------------

먼저 앞서 설명한 PBR 모델을 구현하는 프래그먼트 셰이더를 작성해 보겠습니다. 표면 음영 처리에 필요한 관련 PBR 입력값을 가져와야 합니다.

```

#version 330 core
out vec4 FragColor;
in vec2 TexCoords;
in vec3 WorldPos;
in vec3 Normal;
  
uniform vec3 camPos;
  
uniform vec3  albedo;
uniform float metallic;
uniform float roughness;
uniform float ao;

```


우리는 일반적인 정점 셰이더에서 계산된 표준 입력값과 객체 표면 전체에 걸쳐 일정한 재질 속성 세트를 사용합니다.

그런 다음 프래그먼트 셰이더의 시작 부분에서 모든 조명 알고리즘에 필요한 일반적인 계산을 수행합니다.

```

void main()
{
    vec3 N = normalize(Normal); 
    vec3 V = normalize(camPos - WorldPos);
    [...]
}

```


### 직접 조명

이 장의 예제 데모에서는 장면의 조도를 나타내는 총 4개의 점광원이 있습니다. 반사율 방정식을 만족시키기 위해 각 광원을 순회하면서 개별 복사휘도를 계산하고, BRDF와 광원의 입사각으로 스케일링한 각 광원의 기여도를 합산합니다. 이 순회 과정은 직접 광원에 대한 Ω에 대한 적분 Ω를 푸는 것으로 생각할 수 있습니다. 먼저 각 광원에 대한 관련 변수들을 계산합니다.

```

vec3 Lo = vec3(0.0);
for(int i = 0; i < 4; ++i) 
{
    vec3 L = normalize(lightPositions[i] - WorldPos);
    vec3 H = normalize(V + L);
  
    float distance    = length(lightPositions[i] - WorldPos);
    float attenuation = 1.0 / (distance * distance);
    vec3 radiance     = lightColors[i] * attenuation; 
    [...]  

```


선형 공간에서 조명을 계산할 때 (우리는 [감마 보정](https://learnopengl.com/Advanced-Lighting/Gamma-Correction) 셰이더의 끝부분에서, 우리는 물리적으로 더 정확한 역제곱 법칙을 사용하여 광원을 감쇠시킵니다.

물리적으로는 정확하지만, (물리적으로는 정확하지 않더라도) 빛의 에너지 감소를 훨씬 더 세밀하게 제어할 수 있는 상수-선형-2차 감쇠 방정식을 사용하는 것이 좋을 수도 있습니다.

다음으로, 각 광원에 대해 전체 쿡-토랜스 반사 BRDF 항을 계산하고자 합니다.

\\\[ \\frac{DFG}{4(\\omega\_o \\cdot n)(\\omega\_i \\cdot n)} \\\]

먼저 해야 할 일은 표면이 빛을 반사하는 양과 굴절시키는 양의 비율, 즉 정반사와 난반사 비율을 계산하는 것입니다. 우리는 다음을 통해 알고 있습니다. [이전의](https://learnopengl.com/PBR/Theory) 프레넬 방정식이 바로 그것을 계산한다는 장을 참고하세요. `clamp` (검은 반점을 방지하기 위해):

```

vec3 fresnelSchlick(float cosTheta, vec3 F0)
{
    return F0 + (1.0 - F0) * pow(clamp(1.0 - cosTheta, 0.0, 1.0), 5.0);
}  

```


프레넬-슐릭 근사법은 F0이라는 매개변수를 필요로 하는데, 이는 '입사각이 0일 때의 표면 반사율' 또는 표면을 직접 바라볼 때 반사되는 정도를 나타냅니다. F0은 재질마다 다르며, 대규모 재질 데이터베이스에서 볼 수 있듯이 금속의 경우 색조가 다릅니다. PBR 금속 워크플로우에서는 대부분의 유전체 표면이 일정한 F0으로 시각적으로 올바르게 보인다는 단순화된 가정을 사용합니다. `0.04`금속 표면의 경우 F0 값을 알베도 값으로 지정하는데, 이는 다음과 같은 코드로 표현됩니다.

```

vec3 F0 = vec3(0.04); 
F0      = mix(F0, albedo, metallic);
vec3 F  = fresnelSchlick(max(dot(H, V), 0.0), F0);

```


보시다시피, 비금속 표면의 경우 F0은 항상 같습니다. `0.04`금속 표면의 경우, 원래의 F0 값과 금속 특성에 따른 알베도 값 사이를 선형 보간하여 F0 값을 변화시킵니다.

(F)가 주어졌을 때, 계산해야 할 나머지 항은 정규 분포 함수 (D)와 기하 함수 (G)입니다.

직접 PBR 조명 셰이더에서 해당 코드에 해당하는 내용은 다음과 같습니다.

```

float DistributionGGX(vec3 N, vec3 H, float roughness)
{
    float a      = roughness*roughness;
    float a2     = a*a;
    float NdotH  = max(dot(N, H), 0.0);
    float NdotH2 = NdotH*NdotH;
	
    float num   = a2;
    float denom = (NdotH2 * (a2 - 1.0) + 1.0);
    denom = PI * denom * denom;
	
    return num / denom;
}

float GeometrySchlickGGX(float NdotV, float roughness)
{
    float r = (roughness + 1.0);
    float k = (r*r) / 8.0;

    float num   = NdotV;
    float denom = NdotV * (1.0 - k) + k;
	
    return num / denom;
}
float GeometrySmith(vec3 N, vec3 V, vec3 L, float roughness)
{
    float NdotV = max(dot(N, V), 0.0);
    float NdotL = max(dot(N, L), 0.0);
    float ggx2  = GeometrySchlickGGX(NdotV, roughness);
    float ggx1  = GeometrySchlickGGX(NdotL, roughness);
	
    return ggx1 * ggx2;
}

```


여기서 중요한 점은 다음과 같은 점이 대조적으로 나타난다는 것입니다. [이론](https://learnopengl.com/PBR/Theory) 이 장에서는 거칠기 매개변수를 이러한 함수에 직접 전달합니다. 이렇게 하면 원래 거칠기 값에 특정 항을 수정하는 작업을 수행할 수 있습니다. 디즈니의 관찰을 바탕으로 에픽 게임즈가 채택한 방식에 따르면, 기하학적 구조와 법선 분포 함수 모두에서 거칠기를 제곱하면 조명이 더욱 자연스러워 보입니다.

두 함수가 모두 정의되었으므로 반사율 루프에서 NDF와 G 항을 계산하는 것은 간단합니다.

```

float NDF = DistributionGGX(N, H, roughness);       
float G   = GeometrySmith(N, V, L, roughness);       

```


이를 통해 쿡-토랜스 BRDF를 계산하기에 충분한 정보를 얻을 수 있습니다.

```

vec3 numerator    = NDF * G * F;
float denominator = 4.0 * max(dot(N, V), 0.0) * max(dot(N, L), 0.0)  + 0.0001;
vec3 specular     = numerator / denominator;  

```


참고로 저희는 추가합니다. `0.0001` 내적 결과가 0이 되는 경우 0으로 나누는 오류를 방지하기 위해 분모에 추가합니다. `0.0`.

이제 각 광원이 반사율 방정식에 기여하는 정도를 계산할 수 있습니다. 프레넬 값은 \\(k\_S\\)에 직접적으로 대응하므로, F를 표면에 닿는 모든 빛의 정반사 기여도를 나타내는 데 사용할 수 있습니다. \\(k\_S\\)로부터 굴절률 \\(k\_D\\)를 계산할 수 있습니다.

```

vec3 kS = F;
vec3 kD = vec3(1.0) - kS;
  
kD *= 1.0 - metallic;	

```


kS는 반사되는 빛의 에너지를 나타내므로, 나머지 빛 에너지의 비율은 굴절되는 빛의 에너지이며 이를 kD로 저장합니다. 또한, 금속 표면은 빛을 굴절시키지 않으므로 확산 반사가 없기 때문에 표면이 금속인 경우 kD 값을 0으로 설정하여 이 속성을 적용합니다. 이렇게 하면 각 빛의 반사율 값을 계산하는 데 필요한 최종 데이터를 얻을 수 있습니다.

```

    const float PI = 3.14159265359;
  
    float NdotL = max(dot(N, L), 0.0);        
    Lo += (kD * albedo / PI + specular) * radiance * NdotL;
}

```


결과적으로 얻어지는 Lo 값, 즉 방출되는 복사 휘도는 반사율 방정식의 적분 \\(\\int\\)을 \\(\\Omega\\)로 나눈 값입니다. 모든 가능한 입사광 방향에 대해 적분을 계산할 필요는 없습니다. 왜냐하면 프래그먼트에 영향을 줄 수 있는 입사광 방향이 정확히 4개라는 것을 알고 있기 때문입니다. 따라서 이러한 입사광 방향(예: 장면 내 광원의 개수)에 대해 직접 반복문을 사용할 수 있습니다.

이제 남은 것은 직접 조명 결과인 Lo에 (즉흥적인) 주변광 항을 추가하는 것이며, 그러면 조각의 최종 조명 색상을 얻게 됩니다.

```

vec3 ambient = vec3(0.03) * albedo * ao;
vec3 color   = ambient + Lo;  

```


### 선형 및 HDR 렌더링

지금까지 우리는 모든 계산을 선형 색 공간에서 수행한다고 가정했는데, 이를 고려하기 위해서는 다음이 필요합니다. [감마 보정](https://learnopengl.com/Advanced-Lighting/Gamma-Correction) 셰이더의 마지막 부분에서 선형 공간에서 조명을 계산하는 것이 매우 중요합니다. PBR은 모든 입력값이 선형이어야 하기 때문입니다. 이를 고려하지 않으면 잘못된 조명이 생성됩니다. 또한, 빛 입력값이 물리적 값에 최대한 가깝도록 하여 복사량이나 색상 값이 넓은 범위에 걸쳐 크게 변할 수 있도록 해야 합니다. 따라서 Lo 값은 빠르게 매우 높아질 수 있으며, 이후 특정 값 사이에서 제한됩니다. `0.0` 그리고 `1.0` 기본 출력값이 낮은 동적 범위(LDR)이기 때문입니다. 이를 해결하기 위해 Lo 값을 가져와 톤 또는 노출 맵을 적용합니다. [높은 동적 범위](https://learnopengl.com/Advanced-Lighting/HDR) 감마 보정 전 HDR 값을 LDR 값으로 정확하게 변환합니다.

```

color = color / (color + vec3(1.0));
color = pow(color, vec3(1.0/2.2)); 

```


여기서는 라인하르트 연산자를 사용하여 HDR 색상을 톤 매핑함으로써, 매우 가변적인 조도의 높은 동적 범위를 보존한 후 감마 보정을 수행합니다. 별도의 프레임 버퍼나 후처리 단계가 없으므로, 톤 매핑과 감마 보정 단계를 포워드 프래그먼트 셰이더의 끝에서 직접 적용할 수 있습니다.

![OpenGL PBR 렌더러에서 선형 렌더링과 HDR 렌더링의 차이점.](https://learnopengl.com/img/pbr/lighting_linear_vs_non_linear_and_hdr.png)

PBR 파이프라인에서 선형 색 공간과 고해상도(HDR)를 모두 고려하는 것은 매우 중요합니다. 이 두 가지를 고려하지 않으면 다양한 광도의 밝고 어두운 디테일을 제대로 포착할 수 없어 계산이 부정확해지고 시각적으로 만족스럽지 못한 결과가 나타납니다.

### 완전 직접 조명 PBR 셰이더

이제 남은 것은 최종 톤 매핑 및 감마 보정을 거친 색상을 프래그먼트 셰이더의 출력 채널로 전달하는 것뿐이며, 이렇게 하면 직접 PBR 조명 셰이더가 완성됩니다. 참고로 전체 메인 함수는 아래에 나열되어 있습니다.

```

#version 330 core
out vec4 FragColor;
in vec2 TexCoords;
in vec3 WorldPos;
in vec3 Normal;

// material parameters
uniform vec3  albedo;
uniform float metallic;
uniform float roughness;
uniform float ao;

// lights
uniform vec3 lightPositions[4];
uniform vec3 lightColors[4];

uniform vec3 camPos;

const float PI = 3.14159265359;
  
float DistributionGGX(vec3 N, vec3 H, float roughness);
float GeometrySchlickGGX(float NdotV, float roughness);
float GeometrySmith(vec3 N, vec3 V, vec3 L, float roughness);
vec3 fresnelSchlick(float cosTheta, vec3 F0);

void main()
{		
    vec3 N = normalize(Normal);
    vec3 V = normalize(camPos - WorldPos);

    vec3 F0 = vec3(0.04); 
    F0 = mix(F0, albedo, metallic);
	           
    // reflectance equation
    vec3 Lo = vec3(0.0);
    for(int i = 0; i < 4; ++i) 
    {
        // calculate per-light radiance
        vec3 L = normalize(lightPositions[i] - WorldPos);
        vec3 H = normalize(V + L);
        float distance    = length(lightPositions[i] - WorldPos);
        float attenuation = 1.0 / (distance * distance);
        vec3 radiance     = lightColors[i] * attenuation;        
        
        // cook-torrance brdf
        float NDF = DistributionGGX(N, H, roughness);        
        float G   = GeometrySmith(N, V, L, roughness);      
        vec3 F    = fresnelSchlick(max(dot(H, V), 0.0), F0);       
        
        vec3 kS = F;
        vec3 kD = vec3(1.0) - kS;
        kD *= 1.0 - metallic;	  
        
        vec3 numerator    = NDF * G * F;
        float denominator = 4.0 * max(dot(N, V), 0.0) * max(dot(N, L), 0.0) + 0.0001;
        vec3 specular     = numerator / denominator;  
            
        // add to outgoing radiance Lo
        float NdotL = max(dot(N, L), 0.0);                
        Lo += (kD * albedo / PI + specular) * radiance * NdotL; 
    }   
  
    vec3 ambient = vec3(0.03) * albedo * ao;
    vec3 color = ambient + Lo;
	
    color = color / (color + vec3(1.0));
    color = pow(color, vec3(1.0/2.2));  
   
    FragColor = vec4(color, 1.0);
}  

```


바라건대, 함께라면 [이론](https://learnopengl.com/PBR/Theory) 이전 장에서 배운 반사율 방정식을 활용하면 이 셰이더는 더 이상 어렵게 느껴지지 않을 겁니다. 이 셰이더에 4개의 점 광원과 여러 개의 구체를 사용하고, 각 구체의 금속성 값과 거칠기 값을 수직축과 수평축으로 각각 변화시키면 다음과 같은 결과가 나옵니다.

![OpenGL에서 거칠기와 금속성 값이 다양한 PBR 구체를 렌더링한 결과입니다.](https://learnopengl.com/img/pbr/lighting_result.png)

아래에서 위로 금속 함량 범위는 다음과 같습니다. `0.0` 에게 `1.0`왼쪽에서 오른쪽으로 갈수록 거칠기가 증가합니다. `0.0` 에게 `1.0`보시다시피, 이해하기 쉬운 이 두 가지 매개변수만 변경해도 매우 다양한 재질을 표시할 수 있습니다.

데모의 전체 소스 코드는 여기에서 확인할 수 있습니다. [여기](https://learnopengl.com/code_viewer_gh.php?code=src/6.pbr/1.1.lighting/lighting.cpp).

텍스처드 PBR
------------

이제 시스템을 확장하여 표면 매개변수를 균일한 값 대신 텍스처로 받아들일 수 있게 됨으로써, 표면 재질의 속성을 프래그먼트별로 제어할 수 있게 되었습니다.

```

[...]
uniform sampler2D albedoMap;
uniform sampler2D normalMap;
uniform sampler2D metallicMap;
uniform sampler2D roughnessMap;
uniform sampler2D aoMap;
  
void main()
{
    vec3 albedo     = pow(texture(albedoMap, TexCoords).rgb, 2.2);
    vec3 normal     = getNormalFromNormalMap();
    float metallic  = texture(metallicMap, TexCoords).r;
    float roughness = texture(roughnessMap, TexCoords).r;
    float ao        = texture(aoMap, TexCoords).r;
    [...]
}

```


아티스트가 제작한 알베도 텍스처는 일반적으로 sRGB 색 공간으로 만들어지기 때문에 조명 계산에 사용하기 전에 먼저 선형 색 공간으로 변환해야 합니다. 아티스트가 앰비언트 오클루전 맵을 생성하는 데 사용하는 시스템에 따라 이러한 맵도 sRGB에서 선형 색 공간으로 변환해야 할 수 있습니다. 메탈릭 맵과 러프니스 맵은 거의 항상 선형 색 공간으로 만들어집니다.

이전 구체들의 재질 속성을 텍스처로 대체한 것만으로도 기존에 사용했던 조명 알고리즘에 비해 시각적으로 상당한 개선이 이루어졌습니다.
